# Content Pipeline Skill

## 파이프라인 실행 흐름
1. 소재 수집 (collectors) → materials 테이블
2. 소재 점수화 + 중복 제거
3. 전략 디렉터가 소재 선별 + 파이프라인 배정
4. 콘텐츠 라이터가 채널별 변환 실행
5. 브랜드 가디언이 톤/팩트 검증
6. 휴먼라이크 필터로 AI 냄새 제거
7. 퍼블리셔가 채널 API로 발행

## 파이프라인 디버깅 체크리스트
1. 수집 실패: API 키 확인 → rate limit 확인 → 응답 형식 변경 확인
2. 변환 실패: Claude API 응답 Zod 검증 → 프롬프트 토큰 초과 확인
3. 검증 실패: 브랜드 바이블 규칙 확인 → 톤 점수 임계값 조정
4. 발행 실패: 채널 API 인증 확인 → 포맷 규격 확인 → rate limit 확인

## 채널별 포맷 규격 요약
- Medium: 롱폼 2,000~4,000자
- LinkedIn: 인사이트 300~800자
- X Thread: 5~15트윗
- Threads: 500자 이내
- Newsletter: 1,500~3,000자
- Blog: 2,000~5,000자
